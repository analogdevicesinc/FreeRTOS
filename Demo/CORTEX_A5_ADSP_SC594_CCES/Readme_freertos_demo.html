Description

This example illustrates basic use of FreeRTOS and the ADI FreeRTOS add-in on supported SoCs.

Supported Architectures

  • ADSP-SC573
  • ADSP-SC584
  • ADSP-SC589
  • ADSP-SC594

Overview

This demo example implements the official FreeRTOS demo recommendations. It includes example code that demonstrates how to implementand run standard demo tasks, i.e.:

  • LED flash
  • Polled queue tasks
  • Recursive Mutex tasks
  • Blocking Queue tasks
  • Statically allocated tasks
  • Suicidal tasks

Note that the semihosting I/O mechanism, which writes to the CCES console during debug sessions, uses SWI interrupts. This is incompatible with default GCC-compiled I/O code which also uses SWI interrupts. For this reason, stdio function calls initiated on the ARM core are routed out over UART instead and shall be read with a TTY terminal external to CCES. Setup steps are detailed in the TTY Setup section. Also make sure semihosting is disabled in your debug session configuration.

TTY Setup

Connect a Micro USB cable to your PC from the following EZ-Kit UART to USB connector:

  • ADSP-SC573 EZ-Kit: P7
  • ADSP-SC584 EZ-Kit: P7
  • ADSP-SC589 EZ-Kit: P8
  • ADSP-SC594 EZ-Kit: P2

Download, install, and launch your prefered TTY terminal (e.g. PuTTY/TeraTerm) and configure a serial connection with the settings listed in Table 1.

The COM port number will change as you plug USB cables in and out of your PC. It can be found in the Windows Device Manager facility, under the Ports (COM & LPT) collapsible section.

Table 1: COM Settings
Configuration Value
Baud Rate 115200
Data bits 8
Stop bits 1
Parity None
Flow Control None
